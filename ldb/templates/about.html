Architectural basics of our project

Quick information about our project. Here's how it works.

  1. Introduction

    One of the main reasons why this database exists is to have a centralized location of information about liquor, beer, and wine.

    Our idea behind the project was for users to discover new and interesting drinks that they might have never heard of or tried before. Our end goal was to provide an easy to use website that would allow a user to query our database and discover new drinks within our database.

  2. What is this?

    This website is geared towards users who are interested in expanding their knowledge about different styles of liquor, beer, and or wine. This project seeks to allow users to discover these new products along with other different items that could go along with it, such as food.

    The data we have within this project contains many different products that the average user might not know about. Within our demo page, we show how exactly this website could work. We will give several examples with data to show the result of each question.

  3. How was it built?

    paster quickstart -- This project was initially created using the paster quickstart script which is built into TurboGears. This script allows us to generate the code required to begin the project.

    data.py - The main work behind the database file is created in data.py. Each part of the database is known as a class in python. Each class has an init function that takes its attributes and initializes each class. Each class is given a table name so that the project knows what to call the table when other parts of the project references it. Not only does each class have its own attributes, but many of our classes also require foreign keys.

    websetup.py - Within this python file, we enter in all of our setup data. When we compile our project, PasterScript goes through this file and uses DBSession within SQLAchemy to add each value into our tables. First, you add each attribute, using the definition we set up within data.py, and then we use DBSession to add the new created value into the specific table. DBSession uses transactions built into its framework so that if there are errors while generating the database, or if there are erros at run time while users are manipulating the database, DBSession will rollback the DB automatically.

  4. How to compile and run?

    First, you must make sure you are located within the home folder of this project.

    paster setup-app deveopment.ini -- This line will set up the project using data.py and websetup.py. This line will be used when you add new entries into websetup.py. However, for this to work properly you must delete the devdata.db file (assuming it exists). If you do not do this, paster will attempt to add the same data to the DB file and will fail, thus rolling back the changes.

    paster serve development.ini -- This line will compile the server and host the project onto the local host. Once you run this script it will display the URL for where the project is being hosted.
